# ボーカル・ピッチエディタ

アカペラ音声に対し、Melodyne / Auto-Tune Graph Mode 相当の**ピッチ編集**を行う Web アプリ。
WORLD ボコーダーで F0（基本周波数）だけを差し替えて再合成するため、**声質（フォルマント）を保ったまま**音程を編集でき、境界のクリックノイズも原理的に発生しない。

## クイックスタート

**Python 3.9+** さえあれば、リポジトリ直下で 2 ステップ。

```bash
./setup.sh        # 初回のみ: .venv を作り依存を pip で入れる（数分）
./start.command   # 起動: サーバー → ブラウザ(http://localhost:8000/) が開く
```

- macOS では Finder で **`start.command` をダブルクリック**するだけでもよい（`.venv` が無ければ自動でセットアップ）。
- Windows / Linux は `.command` の代わりに:
  ```bash
  source .venv/bin/activate && cd server && uvicorn app:app --port 8000
  ```
- 必要なものは Python 3.9+ のみ（無ければ https://www.python.org/downloads/ ）。
  `pyworld` は多くの環境でビルド済み wheel が入る（ソースビルドになる場合のみ Xcode CLT / build-essential が必要）。

## 主な機能

- **ピッチ編集** — ノートを縦ドラッグで音程変更。50cent スナップ（Shift=1cent 微調整 / Alt=100cent）。
- **分割・結合** — ノートを任意位置で分割し、区間ごとに独立した音程・音量・遷移を設定。境界は smoothstep 補間で滑らかに接続。
- **補正の強さ** — 原音の揺らぎ・ビブラートを残す量を 0〜1 で調整。
- **セグメント音量 / ミュート** — 区間ごとの音量を dB 領域で連続補間（段差なし）。
- **ハモリ** — ノートをコピーし、任意の音程へペーストして別声部を作成（元と同時発音）。
- **範囲選択** — 空き領域のドラッグで複数バーを一括選択（端で自動横スクロール）。`A`+クリックで追加選択。
- **録音** — マイクから生 PCM で録音（通話向け前処理を無効化）。レベルメーター、**メトロノーム（BPM・拍子指定）**、小節線表示。
- **伴奏トラック** — 伴奏を読み込んでアカペラと同時再生（音量・ミュート・ソロ・頭合わせ）。
- **再生ヘッド** — 常時表示。ドラッグで移動、停止位置から再開、↩︎ で先頭へ。
- **出力** — リバーブ・マスターゲイン・トゥルーピークリミッター（-1 dBTP）を通して WAV / FLAC / MP3 で書き出し（ボーカルのみ / 伴奏込みミックス）。
- **その他** — アンドゥ/リドゥ、ズーム、プロジェクト保存（音声 + JSON で編集状態を完全復元）。

## 主な操作

| 操作 | 挙動 |
|---|---|
| セグメント縦ドラッグ | 音程（Shift=1cent / Alt=100cent） |
| `S` | カーソル位置で分割 |
| 分割線ダブルクリック | 結合 |
| `Ctrl`+分割線ドラッグ | 遷移区間長 |
| `G`+縦ドラッグ | セグメント音量 |
| `M` | ミュート切替 |
| 空き領域ドラッグ | 範囲選択 / `A`+クリックで追加選択 |
| `Cmd/Ctrl+C` → カーソルを置いて `V` | ハモリを任意音程で複製 |
| `Cmd/Ctrl+Z` / `Shift+Z` | アンドゥ / リドゥ |

## 設計原則

1. **波形は切らない。** 編集対象は F0 曲線とゲイン包絡のみ。
2. **再合成は全区間を一括。** ボコーダーが位相を連続生成するのでクリックが出ない。
3. **非破壊編集。** 操作は「元 F0 への変換パラメータ」として保持し、いつでも再編集・リセット可能。
4. **音声の実体はサーバー側。** クライアントとは JSON（編集パラメータ）と WAV だけをやり取りする。

## 構成

- **フロント** — Canvas 2D ピアノロール + Web Audio（`server/static/`）。
- **サーバー** — FastAPI + WORLD（`pyworld`）による解析・再合成（`server/app.py`, `server/pitch/`）。

## アプリケーション化について

現状は「ローカルで起動して使う」形。配布・アプリ化したい場合の選択肢:

- **デスクトップアプリ化** — PyInstaller で `.app`/`.exe` に固める、または Electron / Tauri + Python 同梱。利用者は Python を意識せず起動できる。
- **Web サービス化** — サーバーにデプロイし、URL で誰でも利用（Python はサーバー側で動く）。
- **完全ブラウザ単体** — WORLD を WebAssembly へ移植すれば Python 不要になるが、大掛かり。

いずれも音声エンジンが WORLD（C/Python）である点は共通で、Python を「隠す」ことはできても、WASM 化以外で完全に無くすことはできない。
